# 2026-02-21 (Sat) — Matbench Research Strategy Report

> 생성 시각: 2026-02-21 10:39 KST

## 1) 오늘의 목적 (What good looks like)
- **Matbench 리서치의 다음 7일을 움직일 “실험/리포팅 루프”를 확정**한다.
- 결과물이 쌓이도록: (a) 실험 설계 템플릿 (b) 로그/메트릭 스키마 (c) 비교 가능한 베이스라인을 1개 이상 확보.

## 2) 현재 가정 / 리스크
- 가정: 
  - “전략 리포트”는 **실행 가능한 실험 계획 + 측정/로그 기준 + 의사결정 규칙**을 포함해야 가치가 있다.
  - 매일 1회 짧게라도 축적되면, 주간 단위로 자동 회고/우선순위 조정이 가능하다.
- 리스크:
  - 실험을 많이 돌리기 전에 **측정 정의(성공/실패 기준)**이 느슨하면 비교가 불가능해진다.
  - 모델/프롬프트/툴 버전이 섞이면 재현성이 깨진다.

## 3) 이번 주 핵심 질문 (Answer these to unlock progress)
1. **무엇을 ‘좋은 연구 에이전트’로 볼 것인가?**
   - 정확도(정답률) vs 신뢰도(근거/검증) vs 비용/지연 vs 견고성(실패 복구)
2. **어떤 태스크 스위트스팟에서 차별이 생기는가?**
   - 단발 Q&A가 아니라, 멀티스텝(검색→요약→산출물 생성→저장/커밋)에서 성능이 갈릴 가능성
3. **측정 단위는 무엇인가?**
   - run 단위(1회 실행) / episode(여러 턴) / day(일일 생산성) 중 무엇을 기본으로 잡을지

## 4) 오늘 우선순위 (Top 3)
### P0. 메트릭/로그 최소 스펙 고정
- run 메타데이터: 날짜, 코드 해시, 모델, temperature, tool 사용 내역
- outcome: 성공/실패, 실패 타입(네트워크/파싱/권한/추론 오류 등)
- 비용/시간: wall time, 토큰(가능하면), 외부 호출 수
- 산출물: 파일 경로(또는 URL), diff 요약

### P1. 베이스라인 1개 정의 + 재현 가능한 실행 커맨드
- 가장 단순한 “기본 에이전트” (rules 기반 혹은 단일 모델)로 시작
- 동일 입력에 대해 **N회 반복**하여 분산(variance) 확인

### P2. 태스크 묶음(Task suite) 초안 작성
- 연구 성격에 맞는 6~12개 태스크로 구성 (각 태스크는 5~20분 내 완료)
- 예시 카테고리:
  - 문서 리서치(검색→근거 인용→요약)
  - 코드 수정(PR 단위), 테스트 통과
  - 데이터 수집/정리(테이블/스키마)
  - 일정/리마인더/보고 자동화(크론)

## 5) 실행 계획 (Today’s plan, 2~4h 블록)
- (30m) 로그 스키마 초안 문서화 + 파일명 규칙 결정
- (60m) 베이스라인 실행 파이프라인(스크립트/커맨드) 정리
- (60m) 태스크 6개만 먼저 정의(설명, 입력, 기대 출력, 채점 방식)
- (30m) 내일 리포트가 자동으로 비교될 수 있도록 ‘변경점’ 섹션 템플릿 추가

- 로그가 없으면 성과로 인정하지 않는다.
